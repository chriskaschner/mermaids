    This task wires everything together: the selection panel UI, event handlers, color recoloring, undo, and celebration.

    **A. Add triggerCelebration() to sparkle.js**

    Add a new exported function `triggerCelebration(svgRoot)` to sparkle.js that fires multi-burst sparkle effects across the mermaid. Follow the research pattern:
    - 3 burst positions: head (200, 80), body (200, 250), tail (200, 500)
    - Each burst creates 12 particles (circles) in SVG namespace
    - Colors cycle through: gold, #ff69b4, #87ceeb, #dda0dd
    - Each particle has random spread (80px range), random radius (3-8px)
    - Particles get class "sparkle celebration"
    - Stagger bursts with 150ms setTimeout between positions
    - Each particle removed after 1000ms
    - Do NOT modify the existing triggerSparkle() function

    **B. Add initDressUp() to dressup.js**

    Add a new exported function `initDressUp()` that wires event listeners to the selection panel DOM elements created by renderDressUp(). This function is called AFTER the HTML is in the DOM.

    initDressUp() does:
    1. Attach click handlers to .cat-tab buttons: set state.activeCategory, update .active class on tabs, call renderOptions() to repopulate the options row
    2. Create a `renderOptions(category)` internal function that:
       - Clears #options-row innerHTML
       - If category is "tail", "hair", or "acc": creates an .option-btn button for each variant in PARTS[category]. Each button shows a small preview (24x24 SVG thumbnail -- a simplified shape representing the variant, not the full variant). Button gets data-variant attribute. Clicking calls swapPart(category, variantId) and updates .selected class. For "acc", the first option (acc-none) shows an X or "none" icon.
       - If category is "color": creates a .color-swatch button for each color in COLORS. Each swatch is a circle of that color. Clicking calls recolorActivePart(color).
    3. Attach click handler to .undo-btn: calls undo() and refreshes the .selected states on option buttons
    4. Wire checkCompletion() to trigger celebration: modify checkCompletion() to call triggerCelebration() when returning true. Import triggerCelebration from sparkle.js. Only trigger once per completion (track with a `celebrated` boolean that resets if any part is changed back to default).
    5. Call renderOptions("tail") initially to show tail options on load

    **C. Modify renderDressUp() in app.js**

    Update app.js renderDressUp() to build the full dress-up UI:
    1. Fetch the mermaid SVG as before
    2. Build the HTML structure from the research (mermaid-container + selection-panel with category-tabs and options-row)
    3. Insert the fetched SVG into .mermaid-container
    4. Import and call initDressUp() after DOM insertion
    5. Import resetState from dressup.js and call it before init (in case user navigated away and back)
    6. Still call initTouch("#mermaid-svg") for tap sparkle feedback on the mermaid itself

    The selection panel HTML structure (from research):
    - .selection-panel positioned above nav bar
    - .category-tabs row with 4 .cat-tab buttons (tail, hair, acc, color icons as inline SVGs) + 1 .undo-btn
    - .options-row below tabs, populated dynamically by renderOptions()
    - Each tab button has data-category attribute
    - First tab (tail) has .active class by default

    Use the exact inline SVG icons from the research for category tabs (tail fin shape, hair shape, star for accessories, palette circles for color, curved arrow for undo).

    **D. Add selection panel CSS to style.css**

    Add all the CSS from the research code example for:
    - .selection-panel (positioned absolute, bottom: 80px above nav bar, semi-transparent white background)
    - .category-tabs (flex row, centered, 8px gap)
    - .cat-tab and .undo-btn (60x60px min, 14px border-radius, touch-action: manipulation)
    - .cat-tab.active (highlighted border and background)
    - .undo-btn (margin-left: auto to push right)
    - .options-row (flex row, 8px gap, overflow-x auto with -webkit-overflow-scrolling: touch, centered)
    - .option-btn (64x64px min, 14px border-radius, touch-action: manipulation)
    - .option-btn.selected (purple highlight border)
    - .color-swatch (52x52px circle, white border, subtle shadow)
    - .sparkle.celebration animation (celebration-sparkle keyframes: scale 0 -> 1.5 -> 0.3 with translateY, 1s duration)

    Also update .dressup-view to use flexbox column layout:
    - .mermaid-container takes remaining space (flex: 1)
    - .selection-panel sits below it (fixed height)
    - Mermaid SVG should be contained within the mermaid-container without overflow
