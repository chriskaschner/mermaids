**Create 4 coloring page SVGs** in `frontend/assets/svg/coloring/`. Each SVG:
- ViewBox: `0 0 600 800` (portrait, matches iPad aspect)
- Contains 8-15 fillable regions as `<g data-region="name" pointer-events="all">` groups
- Each region has closed `<path>` elements with `fill="white"` and `stroke="#555" stroke-width="2"`
- Non-fillable detail elements (eyes, mouth, small decorative outlines) use `pointer-events="none"` and `fill="none"` with visible stroke
- All fillable regions must be large enough for a 6-year-old (minimum ~60pt equivalent tap target -- at 600px wide viewBox on a 1024px iPad, that means paths should span at least ~36px in SVG coordinates)

**Page themes:**
1. `page-1-ocean.svg` -- Ocean Mermaid: mermaid body, tail, hair, crown, 2-3 seaweed groups, 2-3 fish/bubble groups, water background section
2. `page-2-castle.svg` -- Mermaid Castle: castle tower, castle base, mermaid figure, 2-3 windows/doors, flag, seaweed
3. `page-3-seahorse.svg` -- Seahorse Friend: large seahorse body, seahorse fin, mermaid figure, coral, bubbles, seaweed
4. `page-4-coral.svg` -- Coral Reef: 2-3 coral formations, mermaid figure, fish, starfish, shells, water

**Create `frontend/js/coloring.js`** -- state module mirroring dressup.js structure:

Exports:
- `COLORS` -- same 10-color array as dressup.js (duplicate, not import, per research recommendation: separate state lifecycles)
- `COLORING_PAGES` -- array of `{ id, label, file }` objects for the 4 pages
- `fillRegion(regionGroup, color)` -- gets fillable child elements (path/circle/ellipse/rect with fill !== "none"), captures previous fills, applies new color, pushes undo closure
- `undo()` -- pops last command from undoStack and executes its undo function
- `setSelectedColor(color)` -- sets `state.selectedColor`
- `getSelectedColor()` -- returns `state.selectedColor`
- `resetColoringState()` -- clears undoStack, resets selectedColor to COLORS[2] (hot pink default), clears currentPageId
- `state` -- object with `selectedColor` and `currentPageId`

Internal (not exported):
- `undoStack` array, `MAX_UNDO = 30`
- `pushUndo(fn)` -- pushes `{ undo: fn }`, shifts if over MAX_UNDO
- `getFillableElements(regionGroup)` -- querySelectorAll path/circle/ellipse/rect, filter fill !== "none"

Do NOT wire any DOM event listeners in coloring.js. The module is pure state + operations. Event wiring happens in Plan 02 when app.js builds the coloring UI.
