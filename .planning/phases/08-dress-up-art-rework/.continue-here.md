<decisions_made>

- Switched recoloring from fill-attribute manipulation to CSS hue-rotate filter because the fill heuristic
  (exclude dark/skin/white fills) fundamentally cannot handle diverse skin tones -- mermaid-4 had ZERO
  recolorable elements since all fills were classified as dark, skin, or white
- Removed skin tone color swatches since hue-rotate shifts the whole character uniformly -- separate
  skin tones don't make sense in this model
- hexToHue() extracts hue from swatch hex color, applied as hue-rotate(Xdeg) on the SVG element
</decisions_made>

<context>
Phase 8 uses a flat character gallery (9 diverse mermaids) instead of multi-layer part swapping.
The original fill-based recoloring tried to identify which SVG paths to recolor by excluding outlines
(dark), skin (warm tones), and eyes (white). This broke completely on darker-skinned characters where
ALL fills matched exclusion criteria. CSS hue-rotate is simpler and universal -- it shifts every color
in the SVG proportionally, preserving the character's relative color relationships.
</context>
